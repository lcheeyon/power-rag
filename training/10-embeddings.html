Embeddings

Module 3 · ~7 min read
Two sentences about the same topic will have similar vectors even if they use completely different words — this is the power of semantic search. Embeddings translate meaning into geometry: similar meanings produce vectors that point in similar directions in a high-dimensional space.

What is an Embedding?

An embedding is a dense float array (a vector) that represents the semantic meaning of a piece of text. The embedding model reads the text and outputs a fixed-length array of floating-point numbers — 768 numbers in Power RAG's case.

Two text fragments with similar meaning will have vectors that are close in this 768-dimensional space, measured by cosine similarity. "Car costs" and "automobile pricing" are semantically similar — their vectors will be close even though they share no words.

nomic-embed-text

Power RAG uses nomic-embed-text, an open-source embedding model that runs locally via Ollama. Key properties:

  • 768 dimensions — matches the Qdrant collection configuration
  • No API cost — runs entirely on your machine
  • Strong performance — competitive with commercial embedding APIs on most benchmarks
  • Max input: ~8192 tokens

The Embedding Pipeline

Chunk text "The Q3 revenue increased by 15% year-over-year" │ ▼ nomic-embed-text (via Ollama) │ ▼ float[768] [0.023, -0.145, 0.891, 0.002, -0.334, ...] │ ▼ Stored in Qdrant vector database alongside chunk text and metadata ───────────────────────────────────────────── At query time: User question: "What was Q3 revenue growth?" │ ▼ nomic-embed-text │ ▼ float[768] ← query vector │ ▼ Cosine similarity search in Qdrant Returns top-K most similar chunk vectors

How Spring AI Wires It

The Ollama embedding starter auto-creates an EmbeddingModel bean. VectorStoreConfig receives this bean and passes it to the QdrantVectorStore — which calls it automatically when you add documents or run searches. You never call the embedding model directly in application code.

application.yml — embedding configuration View source ↗ 
spring:
  ai:
    ollama:
      embedding:
        options:
          model: nomic-embed-text
    vectorstore:
      qdrant:
        dimensions: 768  # must match nomic-embed-text output
The dimensions value in the Qdrant config must exactly match the output dimensionality of your embedding model. If you switch embedding models, you must recreate the Qdrant collection with the new dimension count — existing vectors are incompatible.

Why nomic-embed-text Over a Cloud API?

Cloud embedding APIs (OpenAI, Cohere) are excellent but add cost and latency for every document ingested. Since Power RAG is designed as a self-hosted reference implementation:

  • All ingestion happens locally — no per-document API cost
  • Data stays on-premises — important for sensitive documents
  • Ollama makes switching to a different local model trivial (one config change)
